系統簡介

生成式 AI 與 Google Apps Script (GAS) 雲端整合實作

本報告提供一個最完整的「低代碼（Low-Code）」跨平台雲端解決方案。利用 Google Apps Script (GAS) 做為後端，並將大家熟知的 Google Sheets（試算表） 做為極速讀寫的雲端資料庫。 前端介面透過現代化、響應式的 Tailwind CSS 設計，任何人不論在何種電腦或行動裝置上，打開專屬網址，就能看見即時連線的統一資料庫。

管理員權限 (完整)

擁有專屬的後端身分比對密鑰。可透過網頁介面向大模型 (Gemini 3.5 Flash) 提問，並將 AI 生成結果永久新增、更新或刪除入庫。

一般使用者 (唯讀)

無需任何密碼，直接連入網頁即可看見乾淨、條理分明的歷史生成結果卡片群。前端直接封鎖寫入按鈕，安全防範誤寫。

後端防禦核心安全機制

許多開發者誤以為「只要前端網頁把按鈕藏起來就是唯讀」，這其實非常危險。懂技術的人能在瀏覽器控制台手動呼叫後端函數。本系統採 「伺服器端雙重密鑰驗證」，不論前端指令怎麼繞過，後端如果沒有匹配您設定的管理密碼，一律 100% 擋下寫入指令！

STEP 1

步驟一：建立與配置 Google 試算表（雲端資料庫）

要使用 Google 試算表作為資料庫，我們必須定義好欄位結構。請按照下圖指引進行配置：

  1. 在 Google 雲端硬碟建立一份全新的「試算表」，命名為 AI_Cloud_Database.
  2. 將左下角預設的分頁名稱（預設為「工作表1」）點兩下更名為：Data (注意大小寫，前後不要有空格).
  3. 在第一行（Row 1）的儲存格中，手動打入以下欄位名稱：
1 A: ID B: 提問內容 C: AI回應內容 D: 更新時間
2 （留空，由系統寫入） （留空，由系統寫入） （留空，由系統寫入） （留空，由系統寫入）

💡 核心重點：什麼是 SPREADSHEET_ID？該如何取得？

在後端程式碼中，SPREADSHEET_ID 用來告訴後端程式要對哪一個特定的試算表進行讀寫。取得方法極其簡單：

1 打開您剛剛建立好的 Google 試算表 `AI_Cloud_Database`。
2 看一眼您瀏覽器最上方的「網址列」。它的格式通常長得像下面這樣：
https://docs.google.com/spreadsheets/d/1aBcD_efgHiJkLmNoPqRsTuVwXyZ1234567890_XyZ/edit#gid=0
3 夾在 /d/ 之後，以及 /edit 之前那一長串英數與符號混雜的隨機字串（如上圖高亮部分），就是您的 SPREADSHEET_ID。請將其整串複製下來，準備貼入後端代碼中。
重要警告：欄位文字必須絕對一致！

後端程式碼將會嚴格對應試算表第一行的文字。請確保「ID」、「提問內容」、「AI回應內容」、「更新時間」字體、空格皆與程式代碼一模一樣，否則讀取時前端會出現空白或 undefined。

STEP 4

步驟四：部署上線，向世界開放！

完成代碼貼上與修改後，最核心的步驟就是將其作為網頁發布。這是最關鍵、也最容易出錯的一步，請極度仔細執行以下步驟：

1

進入新部署選單

在 Apps Script 編輯器右上角，點選 「部署」 -> 「新部署」。

2

選擇網頁應用程式

點擊左側「選取類型」旁的齒輪圖示，選擇 「網頁應用程式」。

3

極致重要：安全性權限設置

請務必調整這兩項設定：

  • 執行身分 (Execute as)： 選擇 「我 (您的 Google 帳戶)」。
  • 誰有權限存取 (Who has access)： 選擇 「所有人 (Anyone)」。

※ 設定「所有人」代表任何人不需登入您的 Google 帳號就能看到網頁；設定執行身分是「我」，則代表後端一律借用您寫入試算表的權限，達到無縫免登入效果。

4

授予 Google 認證權限

點擊「部署」。首次運行會要求授權，點選「授予存取權限」-> 選擇您的帳戶。若出現警示畫面，點選左下角「進階」-> 點擊「前往 AI_Cloud_Database（不安全）」-> 按下「允許」。

大功告成！

部署完成後，畫面上會出現一串「網頁應用程式 URL」。複製該連結到任何人的電腦 or 手機，即可即時共同存取資料庫囉！

ALERT

實作與運作常見疑難排解

為什麼我寫入資料成功，但網頁卻什麼都讀不到（一片空白）？

這在以前版本是最常見的問題。因為 GAS 預設的 getValues() 會把試算表的時間格轉換成原生 Javascript Date 物件，而 Date 物件完全無法通過網頁底層的傳輸協議發回前端。目前這個版本已經升級為 getDisplayValues()，強制抓取字串，100% 解決了這個問題。如果您依然遇到，請檢查分頁標籤是否真的叫「Data」。

我修改了 Code.gs 或是 Index.html，為什麼重新整理網頁卻沒有變化？

這是 Apps Script 平台最著名的開發大坑。直接重新整理網頁是不適用最新代碼的。每次只要更動任何程式碼，必須點擊右上角：「部署」 -> 「管理部署」 -> 點選「鉛筆 (編輯)」 -> 在版本選單中選擇「新版本」(New Version) -> 按下部署。重新生成的網址（或原網址）才會生效。

Gemini API 連線報錯，或是回覆顯示未授權？

請至 Google AI Studio 確認您的 API 鑰匙未過期且在有效區域。Gemini 3.5 世代目前全面兼容，且具有更大的免費額度，請確保程式最上方的 API KEY 字串沒有填錯。
